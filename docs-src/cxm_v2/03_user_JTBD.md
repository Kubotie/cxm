
# CSM（Customer Success Manager）のJob To Be Done


CSMの役割は、単に顧客の問い合わせに応じたり、利用方法を説明したりすることではありません。CSMは、顧客がPtengineを導入する検討段階から、導入・オンボーディング・日常活用・そして組織内での展開（Expand）に至るまで、顧客の成功ジャーニーの大部分に伴走する存在です。CSMの仕事は、顧客がツールを使える状態になることではなく、**顧客がツールを通じて継続的に成果を出し、その価値が顧客組織内で広がっていく状態を作ること**です。

具体的には、CSMは顧客の導入相談の段階において、顧客の課題や期待を整理し、Ptengineを使うことでどのような成果が実現できるかを顧客と共に設計します。その後、導入段階では計測設計や初期設定などを通じて、顧客が実際にデータやインサイトを得られる状態を作ります。オンボーディングでは、顧客が最初の成功体験を得ることを重視し、ツールの価値を理解し、自ら活用できる状態へと導きます。さらに活用フェーズでは、顧客の業務プロセスの中にPtengineが組み込まれ、継続的な成果改善が行われる状態を支援します。そして最終的には、顧客組織の担当者だけでなく、その上長や経営層、さらには他部署にまで価値が伝わり、利用が自然に拡張していく状態を作ります。

このようにCSMの本質的な仕事は、顧客の成功モデルを設計し、それを顧客組織の中で定着させ、拡張させることです。CSMは顧客の成功の責任を持つ存在であり、顧客がPtengineを通じて継続的に価値を生み出し続ける状態を実現することが、そのJob To Be Doneです。

# GTM（Go To Market / Product Value Enablement）のJob To Be Done

GTMの役割は、開発部門によって作られる製品能力を、顧客が理解し利用できる価値として市場に届けることです。プロダクトは機能として存在していても、その価値が顧客に正しく理解され、適切な形で使われなければ、実際の価値として顧客の中で発揮されることはありません。GTMは、この**製品能力と顧客理解の間に存在するギャップを埋める役割**を担います。

GTMは、顧客が新しい機能や既存機能をどのように理解し、どのような文脈で使うべきかを整理し、それを顧客に伝えるためのストーリーやユースケースを設計します。顧客にとって重要なのは機能そのものではなく、その機能がどのような成果につながるかです。そのためGTMは、製品の設計思想や機能の意図を顧客の課題や業務文脈に翻訳し、顧客がその価値を理解し実際に活用できる状態を作ります。

またGTMは、新しい機能がリリースされた際に、その機能をどの顧客にどのように伝えるべきかを設計し、既存ユーザーに対しては適切な理解と活用方法を届け、これから利用する顧客に対しては製品価値を正しく理解してもらうための説明や導線を作ります。つまりGTMの仕事は、単に製品を紹介することではなく、**製品の能力が顧客の業務の中で価値として発揮される状態を作ること**です。

この意味でGTMのJob To Be Doneは、製品能力を顧客価値へと翻訳し、顧客組織の中でその価値が理解され、利用され、最大化される状態を作ることです。

# SupportのJob To Be Done

Supportの役割は、顧客が製品を利用する過程で発生する疑問や課題を解決することですが、その本質は単なる問い合わせ対応ではありません。Supportは、顧客がツールを利用する中で生じる摩擦や迷いを迅速に解消し、顧客が前進する流れを止めない役割を担っています。またSupportは、Ptmindが掲げる顧客中心文化を、顧客との接点の中で最も直接的に体現する存在でもあります。

顧客がツールを利用する際には、設定方法や機能の理解、データの解釈など、さまざまな疑問や課題が発生します。このとき顧客にとって重要なのは、問題が解決することだけでなく、その解決が迅速であり、かつ自分の状況を理解してもらえているという安心感です。Supportは、顧客の状況や利用状況を理解した上で、最短で問題を解決し、顧客が再び前進できる状態を作ります。

またSupportは、問い合わせを単なる個別対応として終わらせるのではなく、その内容をナレッジとして蓄積し、FAQやガイドの更新、プロダクト改善などにつなげていきます。これにより、同じ摩擦が他の顧客に発生することを防ぎ、顧客体験を継続的に改善していく役割も担います。

このようにSupportのJob To Be Doneは、顧客がツールを利用する過程で発生する摩擦や迷いを迅速に解消し、顧客が前進し続けられる状態を維持すると同時に、顧客中心のブランド体験を実際の顧客接点の中で体現することです。

# CRM（Customer Relationship Management / Lifecycle Automation）のJob To Be Done

CRMの役割は、顧客の行動や利用状況、関心に基づいて顧客を動的にセグメント化し、顧客の状況に適した情報や提案を届けることで、顧客の前進をスケールで支援することです。CSMが直接伴走できる顧客には限りがあるため、CRMはCSMが直接対応できない多数の顧客に対して、適切なタイミングで適切な情報やコンテンツを届けることで、顧客の成功を支援する役割を担います。

CRMの活動はメール配信だけに限られるものではありません。顧客はWebサイト、プロダクト内のメッセージ、メール、コミュニティなど、さまざまなデジタル接点を通じて企業と接触します。CRMはこれらの接点を統合的に捉え、顧客の行動や利用状況をもとに、今その顧客にとって最も有益な情報や提案を届ける必要があります。

例えば、ある機能をまだ利用していない顧客には、その機能の価値や使い方を紹介するコンテンツを届けることが有効ですし、利用が停滞している顧客には活用事例や改善のヒントを届けることが効果的です。また新しい機能がリリースされた場合には、その機能がどの顧客にとって価値があるかを判断し、適切な顧客に適切な説明を届ける必要があります。

CRMの仕事は、顧客の現在の状況や意図に基づいて顧客理解を行い、その理解に基づいた提案やコンテンツを届けることで、顧客が次の行動を取りやすい状態を作ることです。つまりCRMのJob To Be Doneは、顧客の前進をスケールで支援し、CSMの活動と連動しながら、顧客がEngagementに到達するまでのプロセスを加速させることです。

# 最終整理（4ロールの役割）

まとめると、4つのロールはそれぞれ異なる方法で顧客の前進を支援しています。

CSMは、重要顧客に対して深い伴走を行い、顧客組織の中で価値が定着し拡張する状態を作ります。
GTMは、製品能力を顧客価値に翻訳し、顧客が製品を理解し最大限活用できる状態を作ります。
Supportは、顧客の利用の中で発生する摩擦や迷いを迅速に解消し、顧客の前進を止めない役割を担います。
CRMは、顧客の行動や利用状況に基づいて最適な情報や提案を届けることで、顧客の前進をスケールで支援します。

これら4つのロールは独立した活動ではなく、互いに連動しながら顧客の成功を支える構造を形成しています。
CSMが深く伴走する顧客、Supportが瞬間的に支える顧客、CRMがスケールで支援する顧客、そしてGTMが製品価値の理解を促進する活動が組み合わさることで、顧客は導入から活用、そして組織内での拡張へと前進していきます。

そしてCXM Appは、これらの活動を統合し、顧客の前進を組織として継続的に支援できる状態を作るための基盤となります。
